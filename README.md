# 🚀 Neoflex Backend Prep: ООП, Bash, Python, SQL, Git, HTTP и REST

Этот репозиторий создан в рамках подготовки к тесту и участия в курсе по программе "Python Development" в компании **Neoflex**. 

Здесь собраны практические реализации основных требований к кандидату: работа с Linux-терминалом, Git, ООП на Python, проектирование баз данных (SQL) и построение REST API.

## 🛠 Стек технологий
* **Язык:** Python 3
* **База данных:** SQLite3 (SQL запросы, `JOIN`, параметризованные запросы)
* **API клиент:** `requests` (Парсинг JSON, интеграция со сторонними сервисами)
* **API сервер:** `FastAPI` + `Uvicorn` + `Pydantic`
* **Окружение:** Arch Linux, Git

## 📂 Структура проекта и реализованные концепции

1. **Объектно-Ориентированное Программирование (ООП)**
   * `oop_practice.py` — Реализация классов на примере видеокарт. Продемонстрированы **Инкапсуляция** (приватные атрибуты, геттеры/сеттеры), **Наследование** (базовый класс и наследники) и **Полиморфизм** (переопределение методов).

2. **Реляционные Базы Данных и SQL**
   * `db_practice.py` — Проектирование схемы БД, создание таблиц (`CREATE TABLE`), добавление данных (`INSERT`).
   * `select_practice.py` — Чтение данных (`SELECT`) и фильтрация с помощью `WHERE`.
   * `join_practice.py` — Нормализация БД и склеивание нескольких таблиц с помощью `INNER JOIN`.

3. **Сети и Интеграция (ETL процесс)**
   * `api_list.py` — HTTP-клиент (GET-запросы), получение внешних данных по REST API, парсинг и фильтрация JSON массивов.
   * `etl_boss.py` — Полноценный скрипт интеграции (Extract, Transform, Load). Забор данных из внешнего API, их фильтрация и безопасная загрузка в локальную базу данных (с защитой от SQL-инъекций).

4. **Собственный REST API Сервер**
   * `my_api.py` — Бэкенд-сервер на FastAPI. 
     * Отдача данных из БД по GET-запросу в формате JSON.
     * Прием данных через POST-запрос со строгой валидацией входящего JSON через Pydantic.

## 🚀 Как запустить локальный сервер

1. Убедитесь, что установлены `fastapi` и `uvicorn`.
2. Запустите сервер из терминала командой:
   ```bash
   uvicorn my_api:app --reload
   ```
3. API будет доступно по адресу http://127.0.0.1:8000/api/products
4. Документация (Swagger UI) доступна по адресу http://127.0.0.1:8000/docs
